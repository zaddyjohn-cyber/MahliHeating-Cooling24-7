# Mahli Heating & Cooling — website files

Upload the **contents of this folder** to the **root** of your GitHub repo
(`MahliHeating-Cooling24-7`) — the same place the old `index.html` lives.
Do NOT put them inside a new subfolder, or GitHub Pages won't use them.

## Files that must be at the repo root
```
index.html
styles.css
script.js
image-slot.js
.nojekyll
assets/site-bg.mp4          <-- background video (compressed)
assets/hero-logo.png        <-- hero badge logo (top-right)
assets/hero-tech.jpg        <-- hero technician photo
assets/rinnai-install.png
```

## How to update on GitHub (web)
1. Open your repo on github.com.
2. If a `mahli-site` folder got uploaded by mistake, delete it.
3. Delete the old `assets/hero-video.mp4` if it's still there (no longer used).
4. Click **Add file → Upload files**.
5. Drag in `index.html`, `styles.css`, `script.js`, `image-slot.js`, `.nojekyll`.
6. Drag in the whole `assets` folder (must include `site-bg.mp4` and `hero-logo.png`).
7. When asked to overwrite the existing files, confirm yes.
8. Commit. Wait 1–2 minutes, then hard-refresh the page (Ctrl/Cmd + Shift + R).

If you still see the old version, GitHub Pages is just caching — wait a couple
minutes and hard-refresh again.
